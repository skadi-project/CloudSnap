CloudSnap

v1.0.4 — Скриншотер и запись экрана с автоматической выгрузкой в Nextcloud

CloudSnap — приложение для быстрого захвата экрана и записи видео с последующей автоматической отправкой файлов на сервер Nextcloud через WebDAV. Работает из системного трея, поддерживает горячие клавиши, разметку снимков и очередь отправки при отсутствии сети.

Захват / Запись → Разметка (опционально) → Выгрузка WebDAV → Ссылка в буфере

1. Настройка подключения

Подключение к Nextcloud

  1. Откройте окно настроек (двойной клик по иконке в трее или Ctrl+Q для показа)
  2. Введите адрес сервера WebDAV — например https://cloud.example.com/remote.php/webdav/
  3. Введите логин и пароль приложения (создайте его в настройках Nextcloud → Безопасность → Пароли приложений)
  4. Нажмите «Войти и подключить» — приложение проверит соединение
Пароль приложения — не ваш обычный пароль аккаунта. Создайте его в Nextcloud: Настройки → Персональные → Безопасность → Создать пароль приложения.

3. Запись видео

Запуск записи

  • Записать весь экран — через трей, меню или горячую клавишу
  • Записать область — открывается окно выделения, зажмите ЛКМ и выделите прямоугольник
  • При нескольких мониторах — сначала выбор монитора

Индикатор записи

В правом верхнем углу экрана появляется компактная панель (pill) с таймером и кнопками Пауза и Стоп. Pill прижата к правому краю с отступом 10px от границы экрана — одинаково сверху и справа. Размер pill подстраивается под содержимое, окно индикатора остаётся прозрачным за её пределами.

При записи области вокруг выделенной зоны дополнительно появляется красная рамка с отступом 20px от границ области. Рамка не перехватывает клики (setIgnoreMouseEvents) и не отбрасывает тень — чтобы не мешать взаимодействию с рабочим столом.

Управление

ДействиеСпособ
ПаузаКнопка в индикаторе / горячая клавиша записи / меню трея
ПродолжитьКнопка / горячая клавиша записи / меню трея

Формат видео — .mp4. Битрейт и наличие звука настраиваются в параметрах приложения.

Жизненный цикл записи

Под капотом запись управляется state machine с пятью состояниями. Понимание этого полезно, если что-то пошло не так и нужно понять, на каком этапе процесс завис.

СостояниеКогда
idleЗаписи нет. Стартовая точка.
selectingОткрыто окно выбора области. На экране thumbnail выбранного монитора, рисуем прямоугольник выделения.
recordingИдёт запись. MediaRecorder в recorder-окне пишет чанки, таймер на индикаторе тикает, dot пульсирует красным.
pausedЗапись приостановлена. Таймер на индикаторе остановлен, dot стал жёлтым, иконка кнопки сменилась на «Play».
stoppingRecorder финализирует файл: чанки склеиваются, thumbnail генерируется. После этого состояние → idle и файл уходит на выгрузку.

События и переходы

ДействиеПереход
Старт «Записать весь экран»idle → recording
Старт «Записать область»idle → selecting → recording
Паузаrecording → paused
Продолжитьpaused → recording
Стопrecording | paused → stopping → idle
Закрытие приложения при активной записи* → idle через force-stop с уничтожением всех окон
Окно рекордера закрыто вручнуюrecording | paused → idle через emergency-stop с уведомлением
Защита от race conditions. State machine валидирует каждый переход. Невалидный переход (например, повторный Pause после Stop, или Pause до полной загрузки recorder-окна) игнорируется с warning'ом в логе, состояние не меняется. Это страхует от двойных кликов и поздних IPC-сообщений.

4. Горячие клавиши

Три независимых сочетания настраиваются в параметрах:

ДействиеПо умолчанию (Win)По умолчанию (Mac)
Снимок экранаCtrl+Shift+ACmd+Shift+A
Старт записи / Пауза / ПродолжитьCtrl+Shift+VCmd+Shift+V
Остановить записьCtrl+Shift+SCmd+Shift+S
Горячие клавиши не должны конфликтовать друг с другом. При попытке сохранить совпадающие сочетания приложение покажет предупреждение.

Модификаторы: Ctrl+Shift, Ctrl+Alt, Alt+Shift, Cmd+Shift (Mac). Клавиши: A, S, D, X, Z, F, V, R, W, E, Q, T, Space, Esc.

5. Выгрузка и очередь

Автоматическая выгрузка

После снимка или записи видео файл автоматически отправляется на Nextcloud:

  • Успех — ссылка копируется в буфер обмена, запись добавляется в историю
  • Ошибка сети — файл помещается в очередь отправки, приложение повторит попытку при восстановлении сети

Очередь отправки

Файлы из очереди отправляются автоматически при:

  • Получении фокуса любым окном приложения
  • Запуске приложения
Очередь хранится в настройках приложения. Файлы лежат в userData/pending-uploads/. Не удаляйте их вручную — они будут отправлены при следующем запуске.

Публичные ссылки

Если выбран тип ссылки public, CloudSnap создаёт короткую публичную ссылку через Nextcloud OCS Share API. Ссылка доступна без авторизации.

История

Вкладка «История» показывает последние 50 операций. Для каждого файла:

  • Миниатюра для снимков и видео
  • Имя файла, время, статус (загружено / в очереди)
  • Кнопки: скопировать ссылку, открыть в Nextcloud, удалить (из списка)

Устранение проблем

Чёрный или пустой скриншот / видео

На macOS проверьте разрешение Запись экрана: Системные настройки → Конфиденциальность → Запись экрана → добавить CloudSnap. Без разрешения desktopCapturer возвращает пустые thumbnails — и скриншоты, и видео получаются чёрными.

Не работает режим «Окно» или запись области

На macOS требуется разрешение Универсальный доступ: Системные настройки → Конфиденциальность → Универсальный доступ → добавить CloudSnap. Без него нельзя определить границы окон для подсветки и не работает выбор области записи.

Запись «зависла» при закрытии приложения

CloudSnap ловит before-quit в состояниях recording / paused / stopping: сначала останавливает запись корректно (через stop()), и только после финализации файла закрывает приложение. Если окно рекордера было закрыто вручную или разрушено — срабатывает emergency stop с уведомлением «Запись остановлена: <причина>».

Конфликт горячих клавиш

CloudSnap проверяет уникальность трёх сочетаний при сохранении. Если они пересекаются — приложение не даст сохранить и покажет, какие именно совпали. Решение — выбрать разные клавиши.

Где смотреть логи

  • main-процесс — в консоли DevTools (открывается из терминала при запуске через npm start)
  • recorder-окно — в DevTools самого recorder-окна (правый клик → Inspect, если DevTools включены)
  • Задержки старта — строки вида [latency] t=NNNms ... в main-логе. Используются для диагностики задержек от клика до начала записи.
  • Битрейт — строка [bitrate] ... при старте записи показывает применённое значение из настроек.

6. macOS — особенности

Системные разрешения

macOS требует два разрешения для CloudSnap:

РазрешениеДля чегоКак включить
Запись экранаСкриншоты и запись видео (desktopCapturer)Системные настройки → Конфиденциальность → Запись экрана → добавить CloudSnap
Универсальный доступРежим «Окно» (определение границ окон) и запись областиСистемные настройки → Конфиденциальность → Универсальный доступ → добавить CloudSnap
При первом запуске CloudSnap проверяет эти разрешения. Если они отсутствуют, приложение покажет уведомление и предложит открыть нужный раздел системных настроек.

Трей-иконка

На macOS CloudSnap использует иконку tray-icon-mac.png из ресурсов приложения. Иконка отображается в правой части меню-бара.

Автозапуск

На macOS автозапуск настраивается через Login Items. Если включён параметр «Свернуть в трей», приложение запускается скрыто.

Горячие клавиши

На macOS модификатор Cmd (⌘) используется вместо Ctrl. Рекомендуемое сочетание — Cmd+Shift.

Звук при записи

На macOS системный звук захватывается через chromeMediaSource: 'desktop' — это требует разрешения на запись экрана.

О программе

Версия: 1.0.4. Открыть окно «О программе» — в контекстном меню трея, пункт «О CloudSnap». Там ссылки на проект, список используемых библиотек и лицензия.

Стек

  • Electron 28+ — кросс-платформенная обёртка (Windows, macOS)
  • TypeScript 6+ — основной язык проекта
  • electron-store — локальное хранилище настроек и истории
  • webdav 4 — клиент для выгрузки в Nextcloud (загрузка файлов, OCS Share API)
  • mediabunny — работа с медиаданными (генерация миниатюр, медиа-инфо)

Где взять исходники и developer-документацию

  • README.md — в корне репозитория. Инструкции по сборке (Docker / локально), Makefile-команды, требования к окружению, ограничения macOS-сборки в Linux.
  • src/recording/README.md — детальное описание state machine записи (для контрибьюторов: таблица переходов, side-effects, дизайн-решения, примеры тестов).
